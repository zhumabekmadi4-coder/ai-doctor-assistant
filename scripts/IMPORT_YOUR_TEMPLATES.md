# 📦 Импорт ВАШИХ шаблонов в Supabase

## 🎯 Цель
Перенести все ваши существующие шаблоны (с изображениями) из localStorage в Supabase, чтобы они работали после деплоя.

## ⚡ Быстрый старт (3 простых шага)

### Шаг 1: Экспорт шаблонов из браузера (2 минуты)

1. **Откройте рабочую версию приложения** (где у вас есть шаблоны):
   - Если локально: http://localhost:3000
   - Если на Vercel: ваш URL

2. **Откройте файл экспорта** в НОВОЙ вкладке того же браузера:
   ```
   C:\Antigravity\ai-doctor-assistant\scripts\export_templates_from_browser.html
   ```
   
   Или просто перетащите файл в браузер

3. **Страница автоматически найдет ваши шаблоны** и покажет:
   ```
   ✅ Найдено 6 шаблонов от 1 пользователей
   ```

4. **Нажмите кнопку "💾 Скачать файл"**
   - Файл `my_templates.json` будет скачан в папку Downloads

5. **Переместите файл** в папку scripts:
   ```
   Переместите: Downloads/my_templates.json
   В папку: C:\Antigravity\ai-doctor-assistant\scripts\
   ```

### Шаг 2: Обновить структуру БД в Supabase (3 минуты)

**ВАЖНО**: Сначала нужно обновить структуру таблицы templates!

1. Откройте Supabase Dashboard: https://supabase.com/dashboard
2. Выберите ваш проект
3. Перейдите в **SQL Editor**
4. Нажмите **New Query**
5. Скопируйте содержимое файла `update_database_schema.sql`
6. Вставьте в редактор
7. Нажмите **RUN** (или Ctrl+Enter)
8. Дождитесь сообщения: "Структура базы данных успешно обновлена!"

### Шаг 3: Импорт шаблонов в Supabase (1 минута)

## ✅ Проверка результата

### В Supabase Dashboard:

1. Откройте **Table Editor**
2. Выберите таблицу `templates`
3. Вы должны увидеть все ваши шаблоны с:
   - ✅ Названиями
   - ✅ Содержимым
   - ✅ Изображениями (в формате JSON)
   - ✅ Автором (ваш email)

### В приложении:

1. Запустите локальный сервер:
   ```bash
   npm run dev
   ```

2. Откройте http://localhost:3000

3. Войдите в систему

4. Перейдите в раздел "Шаблоны"

5. Вы должны увидеть все ваши шаблоны!

## 🔄 Что происходит с изображениями?

Изображения сохраняются в поле `images` в формате JSON:

```json
{
  "images": [
    {
      "id": "img_123",
      "type": "file",
      "data": "data:image/jpeg;base64,/9j/4AAQSkZJRg...",
      "caption": "Упражнение 1"
    }
  ]
}
```

**Важно**: 
- ✅ Base64 изображения сохраняются как есть
- ✅ URL изображения тоже сохраняются
- ✅ Все работает точно так же, как в localStorage

## 🚀 После деплоя

После деплоя на Vercel/другой хостинг:

1. Ваши шаблоны будут в Supabase
2. Они загрузятся автоматически при входе
3. Все изображения будут отображаться
4. Вы сможете создавать новые шаблоны
5. Они будут сохраняться в Supabase (не в localStorage)

## 🔧 Troubleshooting

### Проблема: Файл my_templates.json не создается

**Решение**:
1. Убедитесь, что вы открыли `export_templates_from_browser.html` в том же браузере, где работает приложение
2. Проверьте консоль браузера (F12) на наличие ошибок
3. Попробуйте кнопку "📋 Копировать JSON" и сохраните вручную

### Проблема: Скрипт импорта выдает ошибку "Could not find column"

**Решение**:
1. Убедитесь, что вы выполнили SQL-скрипт `update_database_schema.sql`
2. Проверьте в Supabase Table Editor, что таблица `templates` имеет все колонки:
   - title
   - content
   - description
   - header_text
   - is_public
   - author_login
   - author_name
   - images
   - created_at
   - updated_at

### Проблема: Шаблоны импортированы, но не отображаются

**Решение**:
1. Проверьте, что вы вошли под тем же email, что и `author_login` в шаблонах
2. Проверьте RLS политики в Supabase
3. Откройте консоль браузера (F12) и проверьте ошибки

### Проблема: Изображения не отображаются

**Решение**:
1. Проверьте, что поле `images` в БД содержит JSON массив
2. Убедитесь, что base64 данные не обрезаны
3. Проверьте размер изображений (Supabase имеет лимиты на размер строки)

## 💡 Альтернативный вариант: Продолжить использовать localStorage

Если вы хотите продолжить использовать localStorage (без миграции в Supabase):

1. **НЕ выполняйте** импорт шаблонов
2. Шаблоны останутся в localStorage браузера
3. Они будут работать локально
4. **НО**: После деплоя на другом устройстве их не будет

**Рекомендация**: Лучше мигрировать в Supabase для:
- ✅ Доступа с любого устройства
- ✅ Резервного копирования
- ✅ Синхронизации между устройствами
- ✅ Возможности делиться шаблонами
